## 🧐 什么是这玩意儿？

这是一个**正经的协议转换网关**，外表看起来是个毫无感情的 `POST /v1/chat/completions` 标准接口，但它的背后连接着千千万万水群的真实网友。

当你从各种大模型客户端（如 NextChat）发送一条消息时：
1. 本网关会把这条消息悄悄扔进你指定的 QQ 群或私聊里。
2. 然后网关会搬个小板凳，**死死盯着群组**（阻塞线程），等待那帮闷骚群友的回复。
3. 一旦有群友（或群里的野生 bot）上钩并回复了你，网关就立马抢走这条消息，包装上 `ChatCompletion JSON` 的外衣，光鲜亮丽地返回给你的客户端。

**简单来说：你在跟 AI 聊天界面说话，但回复你的是真实（且可能很变态）的群友！**

## 🛠️ 食用教程 (只需三步)

### 1. 准备好诱捕笼（NapCatQQ 端配置）
由于我们需要通过 OneBot11 协议对接，请确保你的 NapCat 已经登录。
去 NapCat 的 WebUI 里面进入 **网络配置** -> 添加一个 **正向 WebSocket 服务器 (WsServer)**：
- host: `0.0.0.0`
- port: `3001` (这里随你，下文举例用 3001)
- 勾选启用，保存。

### 2. 架设中转站（运行本网关）
把代码拉下来，进入项目目录，我们为你准备了傻瓜式的**一键爽发脚本**：

**如果你用的是 Linux 服务器：**
```bash
chmod +x start.sh
./start.sh
```

**如果你用的是 Windows 物理机：**
```bash
# 直接双击 start.bat 即可原地起飞
```
脚本会自动检查 Node 环境、安装依赖、并在后台（借用 pm2）启动这个闷骚网关。

### 3. 打开 WebUI 进行最后调教
由于配置文件是自动生成的 `.env`，你可能还没配置过你的 QQ 群。
打开浏览器访问：`http://你的服务器IP:9521`

在管理面板里填好：
- **NapCat WS 地址** (例如: `ws://127.0.0.1:3001`)
- **默认群号** (你想调戏哪个群的群友，就填哪个号)
- **API Key** (防白嫖鉴权密钥，默认 `sk-qq-bridge`)
- -> 点击保存配置，并点一次测试连通性，看到“✅”证明部署成功！

## 🧩 如何作为 API 接入客户端 (或者 OneAPI 等网关)

现在，你已经有了一个可以召唤真实群友的 OpenAI API 了。以各种分发客户端（如 OneAPI / NewAPI）或 ChatGPT Next Web 为例：

1. **接口地址 (Base URL)**: `http://你的服务器IP:9520` *(如果用了 OneApi 就填你 OneApi 的地址)*
2. **API Key (Bearer)**: `sk-qq-bridge`
3. **支持模型名称**: 必须手动加上并选用 `qpt-5.4` 或 `qlaude-opus-4-6`。

> ⚠️ **OneApi 用户必看防踩坑**：
> 如果你把这个 Bridge 接到了 One API 上，请务必在你的渠道（Channel）配置里，把 `qpt-5.4` 和 `qlaude-opus-4-6` 当做自定义模型添加进去！否则 One API 会嫌弃它们并发个 `no available channels` 报错。

## 📜 常见问答 Q&A

**Q: 群友如果不理我怎么办？**
**A**: 网关默默等你 `60秒`。如果不理你，它会自动返回超时 504 错误。这说明你的群友可能不够闷骚，或者他们在打游戏。建议在发消息时带上“来点牛马”、“涩图”之类的关键词（*本网关对因言论导致的封号不负任何责任*）。

**Q: 它支持流式输出 (Stream) 吗？**
**A**: **支持！但不完全支持。** 真实的群友是不会像大模型一样一个个字向外吐字（打字机效果）的。所以当你开启 stream 调用时，网关依然会死死等群友完整发完一条消息，然后**一次性**通过流推送到你的客户端。所以看起来就是：加载半天 -> 全盘托出。

**Q: 我想特定捕获某个极其骚气的群友的回复，不想全群乱抓？**
**A**: 懂你。去 Web 控制面板的“白名单用户”里，填上他/她的 QQ 号！网关就只会捕获这个 QQ 的发言来作为 Assistant 的回复，屏蔽其他闲杂人等。
